## 八、版本号与升级

### 命名规范

```
1.0.0+1   → 首发
1.0.1+2   → Hotfix（仅修复）
1.1.0+10  → 小版本（新内容）
2.0.0+100 → 大版本（重大更新）
```

### Build Number
- iOS：每次提交必须递增
- Android：versionCode 每次必须递增

### 灰度策略
- 首发：100% 全量
- Hotfix：100% 全量（紧急）
- 大版本：50% → 100%（监控 24h）

## 九、风险预案

### 应对差评
1. 24 小时内回复每条 1-3★
2. 列出问题 → hotfix 优先级
3. 重要 fix 在更新日志中提及

### 应对崩溃
1. 第一时间复现
2. 紧急 hotfix（< 48 小时）
3. App Store 加急审核（Expedited Review）

### 应对下架
1. 邮件咨询审核员明确原因
2. 修复后重提
3. 必要时上诉
